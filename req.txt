## 1) Purpose / Goal (What this integration must achieve)

### Primary objective

*   Implement a **connector at VicGrid** to provide **employee-related data** from **SAP SuccessFactors Employee Central (EC)** to the **Government Identity Provisioning System (GIPS)**, to support **Identity Lifecycle Processes** and **IT account provisioning**. [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)

### What SuccessFactors must provide to GIPS

SuccessFactors (via Integration Suite) will transfer to GIPS:

*   **Employee Data**
*   **Location Data**
*   **VicGrid Organisational Structure** [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)

### What GIPS maintains back in SuccessFactors user profile

GIPS will maintain the following in the **SuccessFactors user profile**:

*   **User Id**
*   **Email Address** [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)

### 6.2 Employee selection criteria (Full and Delta rules)

#### Full extract selection (who must be included)

Employee selection includes: [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)

*   **New hires** (Start Date in the **future**)
*   **Terminated users** (Termination Date in the **past**)
*   **Current employees**

#### Delta extract selection (who must be included)

For deltas, identify employees who meet any of these conditions since last run: [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)

*   Changes in **any fields listed** in Employee Data effective as of the **last run date**
*   **Manager change** since last run date
*   **Job changes or terminations** since last run date with rules:
    *   If effective date is **future** → **do not extract now**
    *   If effective date is **current/past** → **extract now**
*   **Hiring event**: data **will be extracted for future hires** (explicit exception vs general “future do not extract” rule) [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)
*   If hiring/termination happened in past and effective change date equals last run date → **extract** [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)

## 10) End-to-end operational flow (What happens in what order)

### A) Cutover / initial load

1.  GIPS calls **Org Structure API** in **Full mode** → refresh full structure in GIPS. [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)
2.  Ensure org structure exists in GIPS **before** running employee extracts. [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)
3.  GIPS calls **Employee API** in **Full mode** → includes future hires, current employees, and terminated users. [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)
4.  GIPS calls **Location API** (selection = effective today). [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)

### B) Daily/ongoing delta operations

1.  GIPS calls Org Structure in **Delta mode** (subject to restructure/SR constraints). [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)
2.  GIPS calls Employee API in **Delta mode** using **ExtractFromDate** with the last stored timestamp; apply delta selection rules (including special future-hire logic). [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)
3.  GIPS calls Location API as needed (not explicitly defined as delta vs full; but selection is “effective today”). [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)

### C) Identifier writeback (inbound)

1.  GIPS generates **UserID / Email / WoVG Id (and SoE Id)**. [\[VicGrid_IA..._Connector | PDF\]](https://people.ey.com/personal/namala_tharun_kumar_gds_ey_com/Documents/Microsoft%20Copilot%20Chat%20Files/VicGrid_IAM_SF_GIPS_IntegrationSuite_Connector.pdf)
